## SW — `openamr-platform-sw` (6 PRs, in order)

### PR 1 — perception (base)
- Branch: `feature/perception-scan-body-filter` (3 commits)
- Title: **feat(perception): real-robot LiDAR body filter + camera bring-up + 15 fps cap**
- Body:
  > Real-robot perception layer: LiDAR self-view body filter tuned for the chassis footprint,
  > camera bring-up, and a 15 fps cap on the IMX708 (`FrameDurationLimits`) to cut Pi load and
  > match the detector's rate. Validated on hardware.

### PR 2 — real bring-up (depends on perception)
- Branch: `feature/real-bringup` (5 commits)
- Title: **feat(bringup): real-robot data-source composition + nav-only goal relay**
- Body:
  > Real-robot bring-up: micro-ROS agent + RPLIDAR + EKF + static TFs, a top-level
  > `bringup.launch.py` (pick sim or real) with `use_camera` / `use_docking` switches, and a plain
  > `/goal_pose → /goal_pose_nav` relay for nav-only operation (exactly one goal forwarder per
  > profile). Merge **after** perception.

### PR 3 — nav2 tuning
- Branch: `feature/nav2-real-tuning` (2 commits)
- Title: **feat(nav2): real-robot navigation tuning + planner speedup**
- Body:
  > Shared `nav2_params.yaml` tuned on hardware, plus a planner speedup (costmap downsampling for
  > SmacPlanner2D, capped planning time, tighter progress checker) to cut time-to-first-motion on
  > the Pi.

### PR 4 — docking (depends on perception; pairs with vision)
- Branch: `feature/docking-apriltag-gate` (3 commits)
- Title: **feat(docking): on-demand AprilTag gate + robot-frame dock approach**
- Body:
  > On-demand AprilTag gate (detector idle during navigation → frees the Pi), throttled to the
  > detector's capacity. Plus a docking rework that computes the dock normal in the **robot
  > (base_link) frame** instead of the wobbly map frame (robust to one outer tag dropping out),
  > sigma-delta PWM so sub-stiction yaw corrections execute, and measured velocity floors.
  > Validated on hardware.

### PR 5 — vision composition (NEW; depends on camera + gate + bringup)
- Branch: `feature/vision-composition` (1 commit)
- Title: **feat(bringup,docking): intra-process vision composition launches**
- Body:
  > Composed launches that run camera + apriltag in one `component_container_mt` with intra-process
  > comms, removing the 3-process / 2-DDS-hop pipeline that starved the detector (5–8 Hz → 15 Hz,
  > vision load ~124% → ~55% on the Pi). Runtime-depends on the camera, apriltag-gate and
  > real-bringup packages — **merge PRs 1, 2, 4 first**.

### PR 6 — diagnostics (independent)
- Branch: `feature/diagnostics` (1 commit)
- Title: **feat(tools): hardware diagnostics scripts + troubleshooting log**
- Body:
  > Hardware diagnostics scripts + a troubleshooting log for the real robot.

**SW merge order:** 1 perception → 2 real-bringup → 3 nav2 → 4 docking → 5 vision-composition; 6 diagnostics anytime.
