## SW — `openamr-platform-sw` (3 PRs, in order)

### PR 1 — real-robot bring-up (base)
- Branch: `feature/real-robot-bringup` (5 commits)
- Title: **feat: real-robot bring-up — perception, drivers, nav2 tuning, vision composition, diagnostics**
- Body:
  > Everything that makes the real robot run and perceive, **except docking**:
  > - **perception**: LiDAR self-view body filter tuned for the chassis + camera bring-up + a 15 fps
  >   cap on the IMX708 (`FrameDurationLimits`) to cut Pi load and match the detector's rate.
  > - **bring-up**: micro-ROS agent + RPLIDAR + EKF + static TFs, a top-level `bringup.launch.py`
  >   (sim/real) with `use_camera` / `use_docking` switches, and a plain `/goal_pose → /goal_pose_nav`
  >   relay for nav-only operation.
  > - **nav2**: `nav2_params.yaml` tuned on hardware (RotationShimController, accel/decel raised past
  >   the motor stiction floor so in-place rotation actually starts) + a planner speedup (costmap
  >   downsampling for SmacPlanner2D, capped planning time, tighter progress checker).
  > - **vision composition**: launches that run camera + apriltag in one `component_container_mt`
  >   with intra-process comms (5–8 Hz → 15 Hz, vision load ~124% → ~55% on the Pi).
  > - **diagnostics**: hardware diagnostics scripts + a troubleshooting log.
  > All validated on hardware.

### PR 2 — docking (merge LAST)
- Branch: `feature/docking` (16 commits)
- Title: **feat(docking): on-demand AprilTag gate + robot-frame approach + reworked NEAR corrector**
- Body:
  > The full docking stack, meant to land **after** PR 1:
  > - On-demand AprilTag gate (detector idle during navigation → frees the Pi), throttled to the
  >   detector's capacity.
  > - Dock normal computed in the **robot (base_link) frame** instead of the wobbly map frame
  >   (robust to one outer tag dropping out), sigma-delta PWM so sub-stiction yaw corrections
  >   execute, measured velocity floors.
  > - **NEAR-field corrector rewrite (2026-07-07, validated on hardware "c'est aligné")**: real
  >   elapsed-time derivative (was a fixed period → jerk on tag reacquisition), fixed-lookahead depth
  >   compensation (raw `atan2(X,Z)` blew up as depth shrank), stability-weighted axis averaging.
  > - Continuous autofocus on the camera (`AfMode:2`) — fixes NEAR-field blur (adds on top of PR 1's
  >   fps cap on `camera.launch.py`; merges cleanly).
  > - The pre-rewrite corrector is documented as legacy in `docs/15_legacy_near_approach.md` and the
  >   exact prior code is tagged `docking-legacy-pre-2026-07-07-near-approach` (rollback reference).
  > `visual_servo_kp`/`kd` were re-tuned for the new signal — note in review.
