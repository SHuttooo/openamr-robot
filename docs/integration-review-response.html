2. The headline fix — reactive safety restored

Review, PR2/PR4 P0: “the active real-robot velocity path bypasses the configured velocity smoother and collision monitor.”

Before

  • navigation_launch.py launched controller/planner/BT only.
  • velocity_smoother + collision_monitor were configured in YAML but never started.
  • Controller published straight to /cmd_vel — no reactive collision braking.
  • Symptom on the real robot: it drove into visible obstacles.

After

  • Both nodes added to the lifecycle and launched.
  • Explicit chain: controller → cmd_vel_nav → velocity_smoother → cmd_vel_smoothed → collision_monitor → /cmd_vel.
  • Verified in sim: both nodes active, exactly one final /cmd_vel writer, robot still reaches its goal.
  • Deployed & rebuilt on the Pi.

Files: openamrobot_nav2/launch/navigation_launch.py, package.xml (added nav2_velocity_smoother, nav2_collision_monitor). Evidence: ros2 lifecycle get → active [3] for both; ros2 topic info /cmd_vel_nav|/cmd_vel_smoothed → 1 pub / 1 sub each.

3. The unified launch & goal routing

Review, PR2: single entry point is good; sim goal forwarding was missing.

Before

  • Sim branch did not launch any /goal_pose → /goal_pose_nav forwarder → RViz “2D Goal Pose” silently dropped.
  • Risk of zero or two forwarders depending on how layers were composed by hand.

After

  • Symmetric rule, both profiles: exactly one forwarder — dock_trigger (docking on) or a plain relay (docking off), chosen by use_docking.
  • dock_trigger now has a startup guard that detects & warns on a second publisher (tested: it fires on a duplicate).
  • RViz goal reaches bt_navigator in sim and real.

4. Bonus — firmware encoder velocity precision

Not requested in the review, but it improves odometry/telemetry quality (cross-PR “time and frames”).

Before

  • RPM = integer ticks / fixed ~20 ms window. At 1024 CPR + 50 Hz → ~2.9 rpm resolution.
  • Low-speed velocity came out as a coarse 0 / 3 / 6 / 9 staircase.

After

  • Period method: window closes only on a real edge → dt spans the true inter-edge time → fine resolution at low speed; 150 ms idle ⇒ 0.
  • Built, flashed, agent reconnected; mirrored into the repo (firmware/lib/encoder/encoder.h).
